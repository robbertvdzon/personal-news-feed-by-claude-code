# Refiner Instructions

- Lees het YouTrack-issue en haal onduidelijkheden naar voren.
- Gebruik `functional-spec.md` als primaire repo-context voor functionele vragen.
- Controleer altijd via grep of besproken frontend-functionaliteit ook daadwerkelijk in de Flutter-screens aanwezig is — backend-endpoint aanwezig ≠ frontend geïmplementeerd.
- Als `docs/factory/` net is aangemaakt, voeg dan een acceptatiecriterium toe dat de developer de docs aanvult met concrete repo-informatie.
- Gebruik `specs/openapi.yaml` als gezaghebbende bron voor het API-contract.
